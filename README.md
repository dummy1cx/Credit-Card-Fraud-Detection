# Credit-Card-Fraud-Detection

Overview:

Credit card fraud is a hot topic when discussing financial transactions and consumer security on digital platforms. With a current market defined by everyday credit card purchases, the rise of different types of fraud is inevitable, which raises concerns in  large markets about the need for sophisticated fraud detection and prevention. Considering these perspectives, the Credit Card Fraud Detection dataset was chosen to be used in order to understand the process behind this application pattern used by global companies.The objective of this coursework is to develop a neural network for credit card fraud detection, understanding the pattern of  fraudulent activities, exploring methods to detect and prevent if any transactions can be labeled as fraudulent transactions with the aim of developing ideas and contributing as a practical approach to this concern.

Understanding the Datasets:

We have source the datasets from Kaggle link : https://www.kaggle.com/datasets/bhadramohit/credit-card-fraud-detection 


In the above datasets, there are a total record of 100,000 credit card transactions which comprises 7 features those are
Transactions ID : A unique ID for identifying each transactions
Transaction Date : The date and time when the transaction occurred  
Amount : The total amount of the transaction
Merchant ID : The merchant where the transactions took place
Transactions Type : Details of the transaction activity (purchase/refund)
Location : The geographic location of the transaction
IsFraud : The target value is a binary classification where 1 denotes the transaction is fraudulent and 0 is for genuine transaction.
